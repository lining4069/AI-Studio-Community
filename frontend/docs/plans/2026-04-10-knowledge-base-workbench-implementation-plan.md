# 知识库工作台实施计划

## 目标

将当前知识库详情页从“可联调骨架”升级为“正式可用的知识库工作台”。

本阶段聚焦四个方向：

- `Overview` 正式配置页
- `Files` 体验补强
- `Retrieve` 正式检索调试页
- `Chat` 正式知识库问答页

## 开发顺序

### Phase 1：Overview 配置页

- 增补知识库更新能力
- 将 `Overview` 从摘要卡片升级为正式表单页
- 补保存反馈、错误提示、字段校验

### Phase 2：Retrieve 正式版

- 重构检索输入区和结果区
- 引入结果卡片、分数、空态和错误态
- 消除调试式展示风格

### Phase 3：Chat 正式版

- 重构为轻量聊天界面
- 区分用户消息与回答消息
- 增加上下文说明、引用片段区域和发送状态

### Phase 4：Files 体验补强

- 优化上传区
- 统一文件状态反馈
- 优化重新索引的交互反馈

### Phase 5：统一收口

- 统一 loading / empty / error 体验
- 页面样式和文案收口
- 完成联调验收

### T007：测试与联调

## 会话执行建议

建议按“一次会话一个明确子目标”推进：

1. 先做 `T001 + T002`
2. 再做 `T003`
3. 再做 `T004`
4. 再做 `T005 + T006`
5. 最后 `T007`

## 完成定义

只有满足以下条件，才能标记“知识库工作台本阶段完成”：

- `Overview / Files / Retrieve / Chat` 四个 tab 都达到正式产品态
- 页面中不再出现明显调试式 JSON 结果展示
- 用户可在单个知识库详情页内完成配置、上传、检索和问答验证
- 关键链路经浏览器联调验证通过
